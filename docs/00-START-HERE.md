# 00 - Start Here

## Explain Like I Am 10

This project is like a learning game for computer science.

Imagine a student wants to learn coding. Instead of reading random notes from many websites, the student opens this app and follows a path:

1. Pick a learning area, like placement, frontend, backend, AI/ML, or interview practice.
2. Pick a subject, like DSA, DBMS, React, Node.js, or Machine Learning.
3. Learn one small topic.
4. Answer quiz questions.
5. Write code.
6. Do an assignment.
7. Earn progress, XP, and badges.

So the app is basically:

> A game-style study app for coding and interview preparation.

## Who Uses It?

Students who are preparing for:

- College placements
- Coding interviews
- Frontend developer jobs
- Backend developer jobs
- AI/ML basics
- Technical and HR interviews

## What Does The User Do?

The user does not just read theory.

The user:

- Chooses a domain
- Chooses a subject
- Opens a learning journey
- Reads theory
- Solves MCQs
- Writes code in the browser
- Runs code
- Practices SQL
- Takes contests or duels
- Practices interviews with an AI interviewer

## What Result Does The User Get?

The user gets:

- Structured learning path
- Practice questions
- Coding editor
- SQL practice
- AI hints
- Interview preparation
- Profile/progress UI

## The Project In One Line

This project is basically:

> A Duolingo-style AI tutor for computer science, coding, and interview preparation.

## Important Truth

This is a strong MVP/prototype.

It has many features, but some parts are not production-ready yet:

- No real login/signup
- Some stats are demo/fake
- Some validation is weak
- Payment verification is incomplete
- Database schema needs updates

So in interviews, do not say:

> This is a fully production-ready platform.

Say:

> This is a full-stack MVP where I built and integrated the main learning, coding, AI, database, and competition flows.
